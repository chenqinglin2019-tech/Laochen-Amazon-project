# 自动补充数据契约

## Agent 需要知道什么

用户只需要提供西柚“流量得分趋势”Excel 文件夹。商品画像、销量、Woot、上架时间和类目背景由 CLI 自动补充。

不要向用户询问除西柚目录之外的其它数据或配置。

如果 CLI 提示本机 skill 包配置不完整，停止并提示需要补齐本地 skill 配置后重试。不要绕过自动补充数据继续生成完整报告。

## 西柚字段优先级

西柚 Excel 已经提供的字段不得被自动补充数据覆盖：

- 价格、Deal、划线价、Prime 价、Coupon、Promotion、Subscribe、Other。
- Ratings 评分、Ratings 数量。
- 自然/广告流量得分和新增广告活动。

这些字段只按西柚 Excel 计算，不使用其它来源替换。

## 自动补充数据只用于什么

自动补充数据只用于：

- 标题、品牌、上架时间、类目路径、卖家基础信息。
- buy box seller 历史，用于 Woot 秒杀识别。
- 卖家数量趋势。
- BSR / 小类排名趋势。
- ASIN 每日销量预测。
- 子类目背景。

样本内季节性由本地 CLI 用西柚流量和自动补充的日销量聚合，不另行要求用户提供类目季节性数据。

## 字段用途边界

### 商品基础信息

用途：

- 基础画像。
- 上线天数和 360 天风险提示。
- 子类目定位。
- 类目背景补充。

禁止：

- 不覆盖西柚价格、促销、评分和评论字段。

### buy box / Woot 相关信息

用途：

- 识别 Woot 秒杀。
- 展示卖家数量趋势。
- 辅助类目/排名背景。

禁止：

- 不参与价格层、最后成交价、促销动作规则。
- 不参与评价异常规则。

### 每日销量预测

用途：

- `daily sales` 是 7 天窗口有效性评估中的销量数据来源。
- 每日 BSR 可作为背景信息。

禁止：

- 自动补充数据中的价格不参与价格规则。

## 生成报告时的表达

面向用户时使用“自动补充数据”或“销量/画像补充数据”这类表达，不要解释底层实现方式。
